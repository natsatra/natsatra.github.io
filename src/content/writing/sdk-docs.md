I worked with the engineering team to author the SDK reference documentation with CLI installation steps, JavaScript snippets, and product-specific syntax. I tested the snippets in Kissflow's in-product JavaScript editor for accuracy, and provided inline error messages as needed.

These docs follow a docs-as-code workflow end to end: written in Markdown, built with Starlight on Astro, and published via GitHub Pages, so every doc update ships through the same review and deploy pipeline as the product code it describes.
